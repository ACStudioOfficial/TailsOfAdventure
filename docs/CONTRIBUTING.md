# Contributing to Tails of Adventure

Welcome! If you are here, you want to help us build a decentralized, open-source animation series. We value every contribution from fixing a typo in the script to suggesting a complex world-building mechanic. 


### 1. No GitHub Experience? No Problem!
You don't need to be a Senior Dev to help. This file is written specifically, that anyone could understand how to use GitHub

### 2. For Writers and Lore-Masters
* All scripts and world-building must follow our `FORMATTING.md` guidelines.
* We prefer Markdown over any proprietary formats (No .docx, no .pdf).
* Submit your ideas via Pull Requests or discuss them in the Issues section first.


### 3. How to ACTUALLY contribute?
1. If you find a plot hole, a bug in a file, or have a cool idea, then head to the **Issues** tab at the top and click "New Issue." Describe your thought in plain English.
2. You can edit `.md` files directly in your browser:
   - Open the file (e.g., in the `/docs` folder).
   - Click the pencil icon (Edit this file).
   - Make your changes.
   - Scroll down and click Commit changes (think of this as "Save with History").

## Our Tech Stack
To keep the project "Sovereign" and accessible, we only use FLOSS (Free/Libre Open Source Software):
* Markdown (`.md`) only. No Word or PDF files.
* Git 

## The Rules
To keep the "Tails" universe consistent, please follow these guidelines:
1. We do not accept raw AI-generated content (scripts or art) without significant manual oversight and artistic intent. We want a human written text, not generated by a machine.
2. All scripts MUST follow the rules defined in `FORMATTING.md`.
3. Avoid proprietary formats. If it can't be opened in an open-source tool, we can't use it.

## How to Start
1. Check the Issues tab for "Good First Issue" labels for now, when we're settling down.
